# Test: Verifier run creates session log symlinks

Verifies that `fflow verify` symlinks both the verifier's and executor's Claude session JSONL logs into the verifier's run directory, and copies the test plan.

## Background

When `fflow verify` runs, it:
1. Launches a verifier agent (which starts a workflow run via `/fflow:start verifier.workflow.yaml`)
2. The verifier launches an executor agent via `run_agent()`
3. After completion, both Claude session JSONL logs should be symlinked into the verifier's run directory:
   - `verifier-session.jsonl` — the verifier agent's own session log
   - `executor-session.jsonl` — the executor agent's session log
4. A copy of the test plan markdown file should also exist in the run directory.

The run directory is at `~/.freeflow/runs/<run-id>/`. The run ID is generated by verifyCore as `verifier-<timestamp>` and passed via `--run-id` in the prompt.

The agent should run `fflow verify` with the existing `e2e/simple-workflow.md` test plan, then find the verifier run directory (the one created by the **outer** verifyCore, not by any nested verify inside the embedded agent) and check for both symlinks.

IMPORTANT: The `fflow verify` command is long-running (it launches nested agents). The embedded agent MUST wait with a 300s timeout, and if it times out, wait again. Do NOT give up after one timeout.

The fflow CLI is already built at `dist/cli.js` in the fflow directory. All commands should be run from the fflow directory.

## Steps
1. **Run verify**: Run `node dist/cli.js verify e2e/simple-workflow.md --test-dir /tmp/e2e-session-logs 2>&1; echo "EXIT:$?"` from the fflow directory and wait for completion. This command takes 3-5 minutes. If wait() times out, call wait() again until you see output containing "EXIT:".
   - Expected: The command completes with EXIT:0 and produces a test report
2. **Find verifier run directory**: The outer `fflow verify` creates a run with ID `verifier-<timestamp>`. List `~/.freeflow/runs/verifier-*/` sorted by time and pick the NEWEST one. Check if `verifier-session.jsonl`, `executor-session.jsonl`, and a `.md` test plan copy exist using `ls -la`.
   - Expected: Both `verifier-session.jsonl` and `executor-session.jsonl` exist as symlinks in the verifier's run directory, pointing to actual JSONL files under `~/.claude/projects/`. A copy of the test plan (`simple-workflow.md`) also exists.
3. **Verify log contents**: Read the first line of both `verifier-session.jsonl` and `executor-session.jsonl` from the verifier run directory to confirm they are valid JSONL
   - Expected: Both files contain valid JSON lines

## Expected Outcomes
- `fflow verify` creates symlinks for both verifier and executor session logs
- Both `verifier-session.jsonl` and `executor-session.jsonl` exist in the verifier's run directory
- A copy of the test plan markdown file exists in the run directory
- Both session log files are valid JSONL containing Claude session data

## Timeout Strategy

| Phase | Suggested wait timeout |
|-------|----------------------|
| fflow verify completion | 300s (retry wait if timeout) |
| File system checks | 30s |
