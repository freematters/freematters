# Test: Verifier run creates session log symlinks

Verifies that `freefsm verify` symlinks both the verifier's and embedded agent's Claude session JSONL logs into the verifier's FSM run directory.

## Background

When `freefsm verify` runs, it:
1. Launches a verifier agent (which starts an FSM run via `/freefsm:start verifier.fsm.yaml`)
2. The verifier launches an embedded agent via `run_agent()`
3. After completion, both Claude session JSONL logs should be symlinked into the verifier's FSM run directory:
   - `session.jsonl` — the verifier agent's own session log
   - `embedded-session.jsonl` — the embedded agent's session log

The FSM run directory is at `~/.freefsm/runs/<run-id>/`. The run ID is generated by verifyCore as `verifier-<timestamp>` and passed via `--run-id` in the prompt.

The agent should run `freefsm verify` with the existing `e2e/simple-workflow.md` test plan, then find the verifier FSM run directory (the one created by the **outer** verifyCore, not by any nested verify inside the embedded agent) and check for both symlinks.

IMPORTANT: The `freefsm verify` command is long-running (it launches nested agents). The embedded agent MUST wait with a 300s timeout, and if it times out, wait again. Do NOT give up after one timeout.

The freefsm CLI is already built at `dist/cli.js` in the freefsm directory. All commands should be run from the freefsm directory.

## Steps
1. **Run verify**: Run `node dist/cli.js verify e2e/simple-workflow.md --test-dir /tmp/e2e-session-logs 2>&1; echo "EXIT:$?"` from the freefsm directory and wait for completion. This command takes 3-5 minutes. If wait() times out, call wait() again until you see output containing "EXIT:".
   - Expected: The command completes with EXIT:0 and produces a test report
2. **Find verifier run directory**: The outer `freefsm verify` creates a run with ID `verifier-<timestamp>`. List `~/.freefsm/runs/verifier-*/` sorted by time and pick the NEWEST one. Check if `session.jsonl` and `embedded-session.jsonl` exist using `ls -la`.
   - Expected: Both `session.jsonl` and `embedded-session.jsonl` exist as symlinks in the verifier's run directory, pointing to actual JSONL files under `~/.claude/projects/`
3. **Verify log contents**: Read the first line of both `session.jsonl` and `embedded-session.jsonl` from the verifier run directory to confirm they are valid JSONL
   - Expected: Both files contain valid JSON lines

## Expected Outcomes
- `freefsm verify` creates symlinks for both verifier and embedded agent session logs
- Both `session.jsonl` and `embedded-session.jsonl` exist in the verifier's FSM run directory
- Both files are valid JSONL containing Claude session data

## Timeout Strategy

| Phase | Suggested wait timeout |
|-------|----------------------|
| freefsm verify completion | 300s (retry wait if timeout) |
| File system checks | 30s |
